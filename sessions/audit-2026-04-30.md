## 1. Railway deploy failures ‚Äî root cause + fix

- [x] **Error (identical for app, worker, relay):**
  ```
  packages/application/src/flooring/work-orders/files/generate-work-order-file.ts(13,33):
  error TS2307: Cannot find module '@builders/pdf' or its corresponding type declarations.
  ```
- [x] [`packages/application/package.json`](../packages/application/package.json) declares `"@builders/pdf": "file:../pdf"` ‚Äî dep is correctly wired.
- [x] [`packages/pdf/`](../packages/pdf/) exists with `package.json`, `src/index.ts`, and a `tsconfig.json`. Build script: `tsc -p tsconfig.json`. Locally `dist/index.d.ts` exists; on Railway's fresh checkout it does not (no `dist/` is committed).
- [x] **Bug in [package.json](../package.json) build scripts** ‚Äî `build:web`, `build:worker`, `build:relay` all chain:
  ```
  domain ‚Üí lib ‚Üí db ‚Üí application ‚Üí <service>
  ```
  `@builders/pdf` is never built. When `application` runs `tsc`, the `@builders/pdf` types resolve to `dist/index.d.ts` which doesn't exist yet ‚Üí TS2307.
- [x] **One-line fix per script** ‚Äî insert `npm run build --workspace @builders/pdf &&` between `db` and `application`. Three scripts, identical edit.
- [x] **Why it works locally:** `packages/pdf/dist/` was built at some point and never wiped, so `tsc` finds the `.d.ts`. On Railway it's a clean checkout each deploy.
- [ ] **Open question:** are we OK to also stop committing `dist/` to repos and rely solely on the build chain? (Should already be the case ‚Äî `.gitignore` and `dist/` artifacts not committed ‚Äî but worth confirming nothing in CI assumes pre-built dist.)
