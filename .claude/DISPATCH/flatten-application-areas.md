## Intent for this session
Every module directory under `packages/application/src/flooring/` and `packages/application/src/management/` is moved up one level to `packages/application/src/<module>/`, and the same is mirrored under `packages/application/tests/`. The now-empty `flooring/` and `management/` area directories are deleted in both trees, the root barrel `src/index.ts` is rewritten to drop the area segment, and every import that escaped its module (pointing at the staying `list-view/` or `shared/` areas, plus the test imports of `src/`) is fixed. This is purely mechanical: zero behavior change, zero schema change, and `/check-gauntlet` green at the end.

## ⚑ Flags — decisions to make / potential gaps
- ⚑ **git-mv vs plain move.** Prefer `git mv` so history follows each directory. Confirm the user is fine with per-directory `git mv` rather than a filesystem move + re-add.
- ⚑ **Commit granularity.** The tree is intentionally red mid-sequence (moves land before import fixups). Decide with the user whether this is ONE commit at the end (recommended — the tree only typechecks once everything is moved AND rewritten) or a staged sequence. DO NOT COMMIT until the user says so; have the ≤17-word message ready.
- ⚑ **Scripted vs manual rewrite.** WARNING: a blind global find-replace of `flooring/` / `management/` is UNSAFE here. The rewrites are depth-class-sensitive (module-root files drop a different number of `../` than submodule files) and a naive replace can fire on the deceptive `create-property-hub.ts` sibling import that must stay byte-identical. If you script it, script it per depth-class with explicit path lists, not one regex over the tree.
- ⚑ **The deceptive UNAFFECTED import.** `src/management/properties/create-property-hub.ts:17-18` import `"../entities/errors.js"` and `"../entities/types.js"`. Because `properties/` AND `entities/` move together (both were under `management/`), this single-`../` same-area sibling reference still resolves after the flatten. It LOOKS like it should change — it must NOT. Leave it byte-identical.
- ⚑ **Test depth-class discipline.** Test imports come in two depth classes and the fix differs: depth-3 (`../../../src/<area>/<module>/...`) and depth-4 submodule tests (`../../../../src/<area>/<module>/<subdir>/...`). Apply the correct drop per class — do not treat them uniformly.
- ⚑ **Asymmetric trees.** `entity-types/` and `user-activity/` have `src/` dirs but NO corresponding `tests/` dirs. That means 16 src moves vs 14 test moves — not a bug, don't go hunting for the "missing" test dirs.
- ⚑ **Out-of-scope stale JSDoc (flag only, DO NOT edit).** `packages/db/src/flooring/inventory/adjustments/locks.ts:10` contains a JSDoc COMMENT referencing `@builders/application/flooring/inventory/adjustments`. After this flatten that path is stale, but it is (a) in another package and (b) a comment, not an import, so it does not affect the build. It is OUT OF SCOPE for this task. Flag it in your final response; do not touch it.
